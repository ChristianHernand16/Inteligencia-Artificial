Algoritmo Minimax¶

El algoritmo de bsqueda Minimax, es una bsqueda de profundidad limitada

El nombre del algoritmo se da al considerar que dada una funcin esttica que devuelve valores en relacin al jugador maximizante, este trata de maximizar su valor mientras que el oponente trata de minimizarlo

Funcionamiento

El funcionamiento de Minimax se define a como se puede elegir el mejor movimiento, dado el caso que el contrincante escoja el peor movimiento.

Realizar el algoritmo de Minimax se realiza en juegos sencillos, ya que implementarlo completamente consumira gran cantidad de tiempo y memoria.

Pasos Minimax

  1. Primero se debe generar todo el nodo y por lo cual uno meta para que se la llegada.

  1. Se realiza el respectivo clculo de los nodos superiores hacia los nodos inferiores donde el fin de conseguir valores mininos y mximos.

Funcionalidades Minimax

  1. Funcionalidades gana, empate y pierda.

Explicar mediante un ejemplo práctico el funcionamiento del algoritmo¶

Supongamos que una empresa quiere realizar una campaa publicitaria.

Se le presentan 3 posibilidades:

Radio 15 minutos de lunes ajueves en un espacio.

TV 1 spot cada semana sobre las 12h

Prensa 1 anuncio 2 das a la semana los lunes y los jueves

Como han hecho campaas anteriormente se han podido valorar los beneficios de las diferentes posibilidades del siguiente modo

Optimista MaxiMax

Es cando unas de las alternativas de la matriz decisin como son las filas le escogemos el mayor valor y por lo cual al final se escoge tambin el ms alto de todos

Optimista MaxiMin

Es cando unas de las alternativas de la matriz decisin como son las filas le escogemos el menor valor y por lo cual al final se escoge tambin el ms alto de todos

Tomando en cuenta una actitud positiva, el medio ms adecuado para realizar la campaa publicitaria sera la radio. En el caso de los pesimistas, la prensa sera el medio ms seguro

# Main 
while run:

    pygame.time.delay(100)

    # Eventos
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            run = False

    # Movimientos con las teclas
    key = pygame.key.get_pressed()
    if key[pygame.K_w]:
        paddle1.rect.y += -paddle_speed
    if key[pygame.K_s]:
        paddle1.rect.y += paddle_speed
    if key[pygame.K_UP]:
        paddle2.rect.y += -paddle_speed
    if key[pygame.K_DOWN]:
        paddle2.rect.y += paddle_speed

    pong.rect.x += pong.speed * pong.dx
    pong.rect.y += pong.speed * pong.dy

    if pong.rect.y > 490:
        pong.dy = -1

    if pong.rect.y < 1:
        pong.dy = 1

    if pong.rect.x > 740:
        pong.rect.x, pong.rect.y = 375, 250
        pong.dx = -1
        paddle1.points += 1

    if pong.rect.x < 1:
        pong.rect.x, pong.rect.y = 375, 250
        pong.dx = 1
        paddle2.points += 1

    if paddle1.rect.colliderect(pong.rect):
        pong.dx = 1

    if paddle2.rect.colliderect(pong.rect):
        pong.dx = -1

    redraw()

pygame.quit()
